# Philosophy

LastLight starts from a refusal: useful intelligence should not require a data center, a cloud account, or a live network connection.

The project favors retrieval over generation because retrieved passages can be inspected, corrected, replaced, translated, and audited by ordinary contributors. In a constrained or dangerous environment, a fluent unsourced answer can be worse than silence.

LastLight therefore treats refusal as a feature. When the local knowledge base is insufficient, the system says so.
